## Context

### Original Request
Fix all findings from a comprehensive 4-lead code review (Architecture, Tech, Security, Performance & Business Logic) for py-vchasno SDK v0.2.0 release.

### Interview Summary
**Key Decisions**:
- **Version**: 0.2.0 — breaking changes allowed under semver 0.x
- **Sync/Async**: Migrate to `unasyncd` (async-first, committed sync code, CI `--check`)
- **Scope**: ALL findings — HIGH + MEDIUM + LOW
- **Test strategy**: Tests after implementation, using existing pytest + pytest-asyncio

**Research Findings (Metis)**:
- `unasyncd` (v0.10+, libcst-based) is superior to `unasync` — handles `asyncio.sleep()→time.sleep()`, `httpx.AsyncClient→httpx.Client` automatically
- **Committed sync code** approach (like elasticsearch-py, psycopg3) preferred over build-time generation for IDE support and code review visibility
- `_http.py` transport layer CAN be auto-generated with `unasyncd` (unlike `unasync` which fails on multi-token replacements)
- Must verify `aclose()→close()` mapping and `Sync`-prefix preservation in PoC

### Metis Review
**Identified Gaps (addressed)**:
- PoC spike needed before full migration — added as Task 1 (gate for all subsequent work)
- `unasyncd` vs `unasync` tool choice — resolved: use `unasyncd`
- Package restructure path `_async/` + `_sync/` — incorporated
- CI `--check` verification for sync freshness — added
- `aclose()→close()` httpx asymmetry — added to PoC validation criteria

## Work Objectives

### Core Objective
Transform py-vchasno from a manually dual-maintained sync/async SDK into a production-hardened, fully typed, async-first SDK with auto-generated sync counterpart.

### Concrete Deliverables
- Restructured package: `_async/` (source of truth) + `_sync/` (generated, committed)
- `unasyncd` configuration in `pyproject.toml` + CI check
- Zero `-> Any` public methods — all 40+ methods properly typed
- Hardened `_http.py`: transport error retry, jitter, HTTPS enforcement, Retry-After cap 60s
- Streaming download API for large files
- Shared utilities: `_files.py` (file upload), `_utils.py` (params collection, ID validation)
- Fixed mypy config (per-line ignores, not global disables)
- Updated tests covering new features and error paths
- Updated CHANGELOG, README, version bump to 0.2.0

### Definition of Done
- [ ] `unasyncd --check` passes in CI
- [ ] `mypy --strict src/` passes with zero `disable_error_code`
- [ ] `ruff check src/ tests/` passes with expanded rule set
- [ ] `pytest tests/ -v` — all tests pass
- [ ] Zero `-> Any` public methods in API surface
- [ ] All HIGH/MEDIUM/LOW review findings addressed

### Must Have
- `unasyncd` async-first architecture
- Typed returns on all public methods
- Transport retry for `httpx.TransportError`
- Jitter in exponential backoff
- HTTPS enforcement on `base_url`
- `Retry-After` capped at 60s
- JSON parsing error handling in `_base.py`
- `download_archive` `with_instruction` parameter
- Streaming download support

### Must NOT Have (Guardrails)
- **NO build-time sync generation** — sync code committed to repo, generated via `unasyncd` dev command
- **NO `unasync`** — use `unasyncd` only (libcst-based, handles all edge cases)
- **NO new external runtime dependencies** — `unasyncd` is dev-only
- **NO changes to Pydantic model field names** — API compatibility preserved
- **NO changes to public API parameter names** (except documented fixes like `download_archive`)
- **NO over-abstraction** — shared utilities are helpers, not frameworks
- **NO premature optimization** — streaming is opt-in alongside existing eager API
